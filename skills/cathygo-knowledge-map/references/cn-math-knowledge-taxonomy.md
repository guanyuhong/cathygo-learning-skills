# CN Math 主知识点层级规范

本文件用于约束 `cn-math-2022` 及后续中国数学课程标准知识图谱的主知识点输出。前台知识图谱应以稳定、可持续学习的主知识点为核心，而不是直接展示课标目标句、学业要求句、评价证据或教学提示。

## 输出层级

默认层级为：

```text
数学
  -> 领域
    -> 主题
      -> 知识点组
        -> 主知识点
          -> 年级进阶 / 课标目标 / 学习证据 / 活动素材
```

说明：

- `领域`：课程标准中的四大领域。
- `主题`：课程标准中的学习主题。
- `知识点组`：用于聚合相近知识点的稳定概念族，是前台树形导航的叶子层。
- `主知识点`：学生需要持续学习、复习、诊断和迁移的核心概念、方法、规则或模型。
- `年级进阶`：不要作为分类树层级；应作为主知识点的属性或详情。
- `课标目标`、`学习证据`、`教学提示`：默认不进入主图，只挂到主知识点详情或内容生产视图。

## 叶子节点合并规则

知识点组需要适度聚合，不要过细。判断规则：

- 如果一个叶子下只有 1-2 个主知识点，优先并入相邻知识点组。
- 如果多个叶子共享同一学习任务、前置关系和评价方式，应合并。
- 如果一个叶子可以自然拆出 5 个以上主知识点，可以保留为独立知识点组。
- 年级不是叶子节点；同一知识点跨年级的内容用 `progression` 表达。
- “形成意识”“增强能力”“能解决问题”等目标性短语不是知识点组。
- 综合与实践的叶子应以活动能力或项目流程组织，不按普通概念硬拆。

## 节点类型

推荐使用以下概念层：

- `subject`：数学。
- `domain`：四大领域。
- `theme`：课标主题。
- `knowledge_group`：知识点组。
- `knowledge_point`：主知识点。
- `progression`：主知识点在某学段/年级的学习进阶。
- `objective`：课标内容要求或学业要求。
- `evidence`：可观察的学习证据或诊断依据。
- `activity`：综合实践、项目学习或应用情境。
- `competency`：核心素养目标，如数感、空间观念、数据意识。

前台主图默认显示 `domain`、`theme`、`knowledge_group`、`knowledge_point`。默认隐藏 `objective`、`evidence`、`teaching_hint`，只在详情面板或审核模式展示。

## 关系类型

主图优先使用：

- `part_of`：主知识点属于知识点组，知识点组属于主题。
- `requires`：前置条件。
- `extends`：进阶发展。
- `related_to`：横向关联。
- `applies_to`：知识点应用到活动、项目或真实情境。
- `assessed_by`：知识点由某学习证据评价。
- `supports_competency`：知识点支撑某核心素养。

## 前台展示要求

- 默认树形导航显示到 `knowledge_group`。
- 点击 `knowledge_group` 后，图谱画布展开该组下的 `knowledge_point`。
- 点击 `knowledge_point` 后，详情面板展示年级进阶、前置知识、后续知识、关联知识、课标目标、学习证据和活动素材。
- 全图默认不平铺所有主知识点；应使用领域、主题、知识点组逐级展开。
- `objective`、`evidence`、`teaching_hint` 默认不显示在主图上。

## 质量检查

生成 CN Math 主知识点图谱时必须检查：

- 是否把课标目标句误当成主知识点。
- 是否把“学生能够说明、完成或应用...”误当成主知识点。
- 是否把年级作为分类树层级。
- 是否把 OCR 混入的命题、教材编写、课程实施内容纳入学生知识图谱。
- 是否为每个主知识点提供所属领域、主题、知识点组和至少一个来源引用。
- 是否为跨年级知识点提供 progression，而不是重复创建互不关联的同名节点。
